# Профили специалистов и маппинг MCP-инструментов

В консилиуме «Агент Консалт» каждый виртуальный агент запускается с определенным профилем (ролью). В зависимости от выбранной роли, агенту динамически подключаются только необходимые MCP-серверы.

---

## 1. Доступные роли и их специализация

1.  **`programmer`** (Программист-разработчик)
    *   *Фокус*: Чистый код (SOLID/DRY), автотесты (Unit/E2E), рефакторинг, баг-фикс.
    *   *Доступные МЦП*: `gitnexus`, `repowise`.
2.  **`web_architect`** (Веб-архитектор / UX & SEO)
    *   *Фокус*: Структура страниц, адаптивность, UX/UI, Core Web Vitals, доступность (WCAG 2.1 AA), технический SEO (Schema.org).
    *   *Доступные МЦП*: `gitnexus`, `repowise`, `vue-docs`, `shadcn`, `nuxt-ui`.
3.  **`system_architect`** (Системный архитектор инфраструктуры / DevOps)
    *   *Фокус*: Сети, серверы, CI/CD, балансировка, Docker, Kubernetes, мониторинг (Grafana, Prometheus).
    *   *Доступные МЦП*: `gitnexus`, `repowise`, `postgres`.
4.  **`app_architect`** (Архитектор распределенных приложений)
    *   *Фокус*: Микросервисы, DDD, проектирование баз данных (репликация, шардирование по Клеппману), интеграции (RabbitMQ/Kafka).
    *   *Доступные МЦП*: `gitnexus`, `repowise`, `postgres`.
5.  **`marketer`** (Маркетолог-стратег)
    *   *Фокус*: Позиционирование (Райс/Траут), УТП, воронка AARRR, SWOT-анализ, сегментация ЦА (JTBD).
    *   *Доступные МЦП*: `perplexity` (для поиска актуальной рыночной информации во внешней сети в реальном времени).
6.  **`general`** (Универсальный консультант)
    *   *Фокус*: Первичный анализ, декомпозиция задач, маршрутизация по специалистам, сравнение альтернатив.
    *   *Доступные МЦП*: `gitnexus`, `repowise`.

---

## 2. Логика ограничений MCP-серверов

Каждая роль имеет жестко ограниченный список МЦП-серверов, прописанный в конфигурационном файле `.claude.json` в домашней папке агента. 
*   **Безопасность**: Агенты не могут получить доступ к инструментам, не относящимся к их компетенции.
*   **Экономия токенов**: Модели не тратят ресурсы контекста на анализ схем нерелевантных инструментов.
*   **Изоляция**: Любой МЦП-сервер запускается с ограниченными правами доступа на чтение (`read_file`) только в границах текущей рабочей папки (`WORKSPACE_ROOT`). Запись файлов проекта дочерним агентам запрещена (режим Read-Only).
