# Навык: Методология TDD (Test-Driven Development)

Методология разработки через тестирование позволяет создавать надежный, тестируемый и модульный код с минимальным количеством багов.

## 1. Цикл Red-Green-Refactor
1. **Red**: Напишите модульный тест, который проверяет новое поведение и падает (так как кода еще нет).
2. **Green**: Напишите минимально необходимое количество кода, чтобы тест прошел успешно. Не заботьтесь о красоте кода на этом этапе.
3. **Refactor**: Проведите рефакторинг кода: устраните дублирование, выделите методы, улучшите имена переменных. Убедитесь, что все тесты по-прежнему зеленые.

## 2. Пирамида тестирования
* **Unit-тесты (70%)**: Быстрые тесты для изоляции бизнес-логики и чистых функций. Все внешние зависимости (БД, API, сеть) должны быть сымитированы (mock/stub).
* **Integration-тесты (20%)**: Проверка интеграции нескольких компонентов (например, API эндпоинт + БД).
* **E2E-тесты (10%)**: Медленные сквозные тесты в реальном или безголовом браузере (Playwright), имитирующие действия пользователя от начала до конца.

## 3. Best Practices
* Один тест должен проверять только одно логическое утверждение (Single Assertion Rule).
* Структурируйте тесты по паттерну AAA (Arrange - Act - Assert).
* Избегайте логики в тестах (циклы, условия) — тесты должны быть максимально простыми.
