# Использование GitNexus MCP-инструментов

Вы можете использовать следующие инструменты (tools), предоставляемые сервером `gitnexus`, для понимания структуры кодовой базы и связей:

- `list_repos`: Вызов без аргументов для получения списка всех проиндексированных репозиториев.
- `context(symbolName: string)`: Получение 360-градусного представления для класса, функции или интерфейса (показывает, кто их вызывает и где они объявлены).
- `query(queryText: string)`: Интеллектуальный поиск по коду для выявления execution-flow (цепочек выполнения), связанных с концепцией.
- `impact(symbolName: string, depth: number)`: Определение blast radius (зоны влияния) — какие функции или модули сломаются, если вы измените указанный символ.
- `detect_changes`: Анализ текущих изменений (git diff) для выявления их влияния на кодовую базу.

**Важно**: Всегда начинайте исследование незнакомого кода с вызова `list_repos` и изучения структуры связей символов через `context` и `query`, прежде чем читать сырые файлы.
