# Навык: OWASP Top 10 & Безопасная разработка

Этот навык описывает ключевые уязвимости веб-приложений по классификации OWASP Top 10 и методы их предотвращения на уровне кода.

## 1. Инъекции (SQL, NoSQL, OS command injection)
* **Проблема**: Модель данных смешивается с пользовательским вводом без экранирования.
* **Решение**: 
  - Всегда используйте параметризованные запросы (prepared statements).
  - Никогда не конкатенируйте строки пользовательского ввода при построении SQL запросов.
  - Используйте ORM/Query Builder (например, Prisma, TypeORM), которые автоматически параметризуют запросы.

## 2. Небезопасная аутентификация и сессии
* **Проблема**: Слабые алгоритмы шифрования паролей, хранение сессий в открытом виде.
* **Решение**:
  - Пароли хэшировать с использованием устойчивых алгоритмов (argon2, bcrypt).
  - Токены сессий должны иметь флаги `HttpOnly`, `Secure`, `SameSite=Strict`.

## 3. Раскрытие чувствительных данных (Sensitive Data Exposure)
* **Проблема**: Передача ПДн (персональных данных) или секретов в открытом виде, логирование паролей или API-ключей.
* **Решение**:
  - Шифровать чувствительные данные при передаче (HTTPS) и хранении.
  - Не хранить секреты в репозитории. Использовать `.env` переменные, защищенные менеджеры секретов (Vault, GCP Secret Manager).
  - Исключать пароли и ключи из логов с помощью маскирования.

## 4. XSS (Cross-Site Scripting)
* **Проблема**: Внедрение вредоносного JS-кода на веб-страницы.
* **Решение**:
  - Экранировать все данные перед рендерингом в HTML (браузер по умолчанию экранирует данные при использовании `textContent`/`innerText`, избегайте `innerHTML`/`dangerouslySetInnerHTML`).
  - Настраивать политику Content Security Policy (CSP).
